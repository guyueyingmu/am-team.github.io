Button 按钮

preview 效果预览

蓝色 按钮 43px 白色按钮 43px 红色43px按钮 最小号按钮

usage 使用说明

  • AntUI默认已集成

  • 模块名：widget/button

  • 可选依赖：util/flexbox（布局，amui包已集成）

使用方式

default 基础样式

  • 基础特征
  • 颜色蓝色(blue)
  • 尺寸高度(43px)，宽度100%
  • 标签建议button或者a
蓝色按钮 43px 
<a href="button" class="am-button" am-mode="blue 43px">蓝色按钮 43px</a>
<button type="button" class="am-button" am-mode="blue 43px">蓝色43px按钮</button>

color 颜色扩展

  • 可选颜色(4种)：
  • 蓝色(mode值：blue，9.0推荐使用)
  • 白色(mode值：white，9.0推荐使用)
  • 红色(mode值：red，9.0推荐使用)
  • 黑色(mode值：black，不推荐)
<button type="button" class="am-button" am-mode="blue 43px">蓝色按钮</button>
<button type="button" class="am-button" am-mode="red 43px">红色按钮</button>
<button type="button" class="am-button" am-mode="white 43px">白色按钮</button>
<button type="button" class="am-button" am-mode="black 43px">黑色按钮</button>

size 尺寸扩展

  • 可选尺寸(5种)：
  • 26px(mode值：tiny)
  • 31px(mode值：little)
  • 36px(mode值：small)
  • 40px(mode值：middle)
  • 42px(default)
  • 43px(新增样式)
<button type="button" class="am-button" am-mode="red tiny">红色26px按钮</button>
<button type="button" class="am-button" am-mode="white little">白色31px按钮</button>
<button type="button" class="am-button" am-mode="black small">黑色36px按钮</button>
<button type="button" class="am-button" am-mode="blue middle">蓝色40px按钮</button>
<button type="button" class="am-button" am-mode="blue">默认42px按钮</button>
<button type="button" class="am-button" am-mode="blue 43px">蓝色43px按钮</button>

disabled 按钮可用性

  • 可选类型(3种)：
  • disabled(通用不可用，最常用)
  • blue-disabled(不推荐，少数场景使用)
  • red-disabled(不推荐，少数场景使用)
<button type="button" disabled="disabled" class="am-button" am-mode="white disabled 43px">白色按钮不可用43px</button>
<button type="button" disabled="disabled" class="am-button" am-mode="blue blue-disabled">特殊不可用蓝色按钮</button>
<button type="button" disabled="disabled" class="am-button" am-mode="red red-disabled">特殊不可用红色按钮 </button>

flat 通栏按钮

<button type="button" class="am-button" am-mode="flat warn 43px">警示按钮</button>
<button type="button" disabled="disabled" class="am-button" am-mode="flat warn 43px">警示按钮不可点43px</button>

light 简易按钮风格

  • 可选类型(3种)：
  • blue(推荐使用)
  • disabled(不推荐，少数场景使用)
<button type="button" class="am-button" am-mode="blue light small">蓝色简易按钮</button>
<button type="button" class="am-button" am-mode="light small disabled">不可用简易按钮</button>

layout 布局

  • 可选依赖：util/flexbox（布局，amui包已集成）
行内按钮 最小号按钮 
<div class="am-flexbox" am-mode="average">
    <div class="am-flexbox-item">
        <button type="button" class="am-button" am-mode="blue">均长按钮</button>
    </div>
    <div class="am-flexbox-item">
        <button type="button" class="am-button" am-mode="white">长短按钮均长</button>
    </div>
</div>
<a href="button" class="am-button" am-mode="white inline">行内按钮</a>
<a href="button" class="am-button" am-mode="tiny light inline">最小号按钮</a>

brief 简述

  • AntUI，原名为AMUI，是Ant Mobile UI的简称。

  • Ta主要有以下特点

    • 支付宝无线端快速建站的解决方案，专为支付宝钱包打造
    • 支持iOS4.2+、Android2.2+版本，为手机端兼容投入大量研究
    • 从钱包8.0开始发展，历经多个版本
    • 有专门的团队支持
    • 服务于支付宝站内、兄弟站点、外部商户接入等
    • 当前版本： AntUI 9.0

Modular 属性选择器和模块化

issue 论点阐述——模块化的契机

  • 属性选择器为模块化带来了契机
    • 解决方案： 一个很小的技术尝试，引入两个目前不太敢使用的选择器，:not()伪类，[xxx]属性选择器。
      • 关于版本控制，比如am-tab组件，他有8.2，8.3，8.4，taobao版，那么仅仅<div class=”am-tab” am-version=”8.2”>，通过am-version控制。
      • 关于不同版本组件并存，同时使用<div class=”am-tab” am-version=”8.2”>，<div class=”am-tab” am-version=”taobao”>，背后的css实现原理基于[xxx]属性选择器，为.am-tab[am-version=”8.2”]，.am-tab[am-version=”taobao”]，各自在各自名称空间（am-version）下写css，无冲突
      • 覆盖即更新，最简单的，我肯定不想带版本，直接这样使用，<div class=”am-tab”>，我不带名称空间写css，.am-tab{}，这好像有点不对，这个样式引入，就会污染.am-tab[am-version=”8.2”]带名称空间的。幸运的是，有:not()伪类可以解决该问题，.am-tab:not([data-version])，标注没有名称空间的情况。
    • 这以前我们使用不多，万一出问题了呢，做技术，必须大胆尝试，小心验证。
      • 看看caniuse上这两个属性兼容性怎么样——哇擦，Android2.1就支持，而且一路标绿，这可比某些橙色部分支持厉害多了。
      • 当然，只看canisue也不行，不放心，我肉身进入我们设备实验室，测了20多款机器，Android2.2，Android2.3，Android4.1，YunOS、iOS、Flyme OS等等系统，小米（红、2）、锤子、三星、摩托罗拉、中兴、vivo、卓普、魅族、HTC等等。等等，一看怎么都还是好的。顺便验证了gradient的三种写法的支持，最老写法-webkit-gradient基本都支持，中间写法-webkit-liner-gradient有些不支持，最新写法liner-gradient基本都不支持。
    • 结果太美好，我们该不该下狠手试试？

why 关于为何引入新方案

  • 方案孵化于AMUI 8.4版，作为探索型新方案，致力于版本并存、覆盖即更新、组件库丰富

  • 目前现状：

    • AMUI的设计风格跟随钱包native设计风格，一个版本改一次，变更频繁，且部分组件，可能需要涉及结构重新改造，更新成本大。
    • 实际使用中，对应业务线都对AMUI的更新换代持谨慎态度，现有已上线产品，本来以稳定上线，AMUI换代，难免产生会引入风险因素的心理。
    • AMUI版本过多，8.0版、8.1版、8.2版、8.3版，大版下面还可能有小版，以实际项目举例，上次天猫包项目中，甚至有使用8.0版AMUI的
    • AMUI自身发展有限，组件有限几个，且被固化在频繁的native风格改版，组件无积累。
    • 同名组件，很容易彼此冲突
    • ……
  • AMUI想做出一些改变，引入组件版本机制，解决版本控制、不同版本组件并存、覆盖即更新。
    • 做到版本控制，小可称为版本控制，大即为风格衍生，同一组件，不同长相，组件库会更丰富，而每一个钱包改版，只是为我们积累更多优秀组件
    • 做到不同版本组件并存，同类组件，如am-tab，但是由8.4版，tabao.xx版，通过版本，作为其同名（am-tab）的额外名称空间，防止冲突。不仅css可以不受限制，html结构也能做到自由灵活。
    • 覆盖即更新，大部分同学使用am-tab，都是希望我引用8.2版的时候，是8.2版，引入8.3版的时候，自然变成8.3版，这我就能特别放心使用amui了。

other 其它

  • 各类升级预案：

    • Old is better：主文件保持老的，部分新样式，指定使用新版本
    • New is better：主文件保持新的，部分老样式，指定使用老版本
    • More is better：无主文件，从丰富的组件库中拼凑出自己的版本。

  • 最后，总结一个，做移动前端真苦逼。不同手机机型，不同系统，都有各种坑。甚至，同个手机上，UC浏览器，原生浏览器等，就还真能表现的不一样。对很多技术，都只能保持保守态度。
    但，我们该大胆尝试和探索。。当然，最好也要多验证，做后备保险，小心使得万年船。。。不然死的还是你。。

rem版AntUI引入 1px问题解决

rem版AntUI能够解决的问题（iOS及Android4.4以上）

  • 1px问题
    • 即H5页面中1px宽的线渲染时在部分高分辨率手机中会占用2px或更高的物理像素。对比native界面显得不精致。

  • Android钱包UI适配问题(在三星S5 Android5机器上有兼容性问题，且视觉不认可，废弃)

    • rem化后，html的font-size大小会根据具体机型的实际宽度与基准宽度（640px）的比例进行缩放。结果是与native的处理保持一致（iOS的native实现没有同比例放大，即高度保持不变，但是宽度会自适应）。
        //默认情况下html的font-size为16px。
    //详细代码请参考下方的行内脚本

试用rem版AntUI的步骤

  • 引用对应的rem版AntUI样式文件

    • 例如，原先引用文件的是<link href="https://a.alipayobjects.com/amui/dpl/9.0.0.2/amui.css" rel="stylesheet"/>
    • 则替换为<link href="https://a.alipayobjects.com/amui/rem/dpl/9.0.0.2/amui.css" rel="stylesheet"/>。
    • 总体替换规则是 /amui/dpl ==》/amui/rem/dpl,9.0.0.1版本以上开始支持

  • 更换viewport的设置方式

    • 原先写死的<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0, user-scalable=0"/>，请先删除或者注释

    • 在header中加入行内脚本，通过脚本来生成viewport的meta信息。脚本的大体思路是先获取当前H5页面的dpr，如果是Android4.4以上机型或者是iOS，则根据dpr的值换算出页面的scale。以iPhone5为例，物理分辨率为640px，scale=1时，H5页面渲染的width则为320px，而当sacle设置为1/dpr=0.5时，H5页面渲染的width则为640px。那么在css中1px就和物理的1px对应上。

    ;(function(win) {
    var docEl = document.documentElement;
    var metaEl = document.querySelector('meta[name="viewport"]');
    var fontEl = document.createElement('style'), dpr,scale,tid;
    if (metaEl) {
        var match = metaEl.getAttribute('content').match(/initial\-scale=(["']?)([\d\.]+)\1?/);
        if (match) {
            scale = parseFloat(match[2]);
            dpr = 1 / scale;
        }
    }
    if (!dpr && !scale) {
        dpr = win.devicePixelRatio || 1;
        scale = 1 / dpr;
    }
    docEl.setAttribute('data-dpr', dpr);
    docEl.firstElementChild.appendChild(fontEl);
    if (!metaEl) {
        metaEl = document.createElement('meta');
        metaEl.setAttribute('name', 'viewport');
        var matches = navigator.userAgent.match(/Android[\S\s]+AppleWebkit\/(\d{3})/i);
        var UCversion = navigator.userAgent.match(/U3\/((\d+|\.){5,})/i);

        if(!matches || matches && matches[1] > 534 || UCversion && parseInt(UCversion[1].split('.').join(''),10) >= 80) {
            metaEl.setAttribute('content', 'width=device-width,user-scalable=no,initial-scale=' + scale + ',maximum-scale=' + scale + ', minimum-scale=' + scale);
            if (docEl.firstElementChild) {
                docEl.firstElementChild.appendChild(metaEl);
            } else {
                var wrap = document.createElement('div');
                wrap.appendChild(metaEl);
                document.write(wrap.innerHTML);
            }
            if(docEl.clientWidth == 980 || docEl.clientWidth == 1024) {
                metaEl.setAttribute('content', 'target-densitydpi=device-dpi,width=device-width,user-scalable=no,initial-scale=1,maximum-scale=1,minimum-scale=1');
            }
        } else {
            metaEl.setAttribute('content', 'width=device-width,user-scalable=no,initial-scale=1,maximum-scale=1,minimum-scale=1');
            if (docEl.firstElementChild) {
                docEl.firstElementChild.appendChild(metaEl);
            } else {
                var wrap2 = document.createElement('div');
                wrap2.appendChild(metaEl);
                document.write(wrap2.innerHTML);
            }

            dpr = 1;
        }
    }

        win.rem = 16 * dpr;
        fontEl.innerHTML = 'html{font-size:' + win.rem + 'px!important;}';

    • 最低控制 用户没有重置类样式时，也能保证基本显示
    • 自由兼容 使用者可以自由额外引入reset or normalize，而不用担心冲突
    • 节约代码 去掉一些不必要的代码，节约是良好的美德
